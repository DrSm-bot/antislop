# Test Corpus Layout

AntiSlop test fixtures are local-first. Do not commit private, copyrighted, customer, or unreleased sample images unless the license and consent are explicit.

The fixture root is `test-corpus/`:

```text
test-corpus/
  README.md
  manifest.example.json
  local/
    human-made/
    ai-generated/
    stripped-metadata/
    transformed/
    c2pa-bearing/
  expected/
```

## Fixture Categories

- `human-made/`: camera, scan, hand-drawn, or exported artwork with known human provenance.
- `ai-generated/`: generated images with generator metadata or known provenance markers.
- `stripped-metadata/`: images intentionally exported, recompressed, screenshotted, or scrubbed so metadata is absent.
- `transformed/`: resized, cropped, compressed, format-converted, or otherwise derived files used to verify marker survival.
- `c2pa-bearing/`: files expected to contain C2PA / Content Credentials bytes.

## Privacy And Licensing

`test-corpus/local/**` is ignored by git. Put real images there for local work only.

If a sample is public-domain, permissively licensed, synthetic, or owned by the project, document the license in the fixture manifest before considering it for commit. Prefer committing tiny generated byte fixtures or expected-result snapshots over real images.

## Manifest Format

Create a private `test-corpus/manifest.local.json` for local files. Use `test-corpus/manifest.example.json` as the template.

Each fixture entry has:

- `id`: stable identifier used for output file names.
- `category`: one of the category directory names.
- `path`: path to a local image, relative to the manifest file.
- `fileType`: MIME type passed to the analysis core.
- `dimensions`: known dimensions, because browser image decoding is not available in the Node runner.
- `expected`: optional expected check states.
- `expectedMetadata`: optional expected structured metadata presence/state.
- `sampleText`: optional Node-compatible fallback bytes for metadata-marker tests without a real image.
- `syntheticXmpDescription`: optional synthetic JPEG/XMP fixture text generated by the runner.
- `license`: required if committing expected snapshots derived from the fixture.

## Expected Snapshots

`test-corpus/expected/**` can be committed when licensing allows. Keep snapshots stable by setting `checkedAt` and `id` in the runner output.

Recommended workflow:

```bash
npm run fixtures:analyze -- --manifest test-corpus/manifest.local.json --write
```

Review the generated JSON before committing it. Expected snapshots should describe what AntiSlop checked, not claim that an image is human-made.

## Node Runner Constraints

The browser app computes image dimensions through `Image.decode()` and object URLs. Those APIs are not available in plain Node, and the project does not currently ship a Node image decoder. The fixture runner therefore exercises the existing analysis core against file bytes and requires dimensions in the manifest, or uses `sampleText` for minimal metadata-marker fixtures.

This is enough to validate hash generation, readable-marker detection, C2PA adapter states, stripped-metadata warnings, and snapshot shape. The Node runner uses synthetic bytes and injected/mocked adapter results for C2PA states; committed real signed C2PA assets are still needed for end-to-end WASM validation fixtures.
